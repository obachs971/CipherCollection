On the Subject of Logical Ternary Manipulation CM

A machine that knows no bounds when it comes to ciphers.

From the Synapse Cipher module by Kilo Bites and VFlyer.

Take the letters on screens 1 and 2 and do the following:

  • Convert the letters into Values via the Value Table.
  • Modulo the values according to the length of the encrypted word (4: 16, 5: 32, 6: 64, 7: 128, 8: 256).
  • Convert the values into binary, adding leading 0s equal to the length of the encrypted word if necessary.

Convert each letter into ternary by using the Ternary Table:

If the Boolean Expression on screen A is true, XOR the first half of the bits, then XNOR the last half of the bits. Otherwise, XNOR the first half of the bits, then XOR the last half of the bits. Concatencate the results into one binary string. However, if the length of the encrypted word is odd, then either XOR the entire binary string if it's true, or XNOR otherwise.

If the Submit Button is white, follow the instructions under Encrypt Instructions. Otherwise follow the instructions under Decrypt Instructions.

Encrypt Instructions

  • If the Xth digit of the resulting binary is a 1, decrement each ternary digit of the Xth ternary value by 1, wrapping around to 2. DO NOT CARRY DIGITS.
  • Otherwise, don't modify the digits.
  • If after incrementing, any 3-digit ternaries are 000, decrement all the digits in 000 by 1 again.

Decrypt Instructions

  • If the Xth digit of the resulting binary is a 1, increment each ternary digit of the Xth ternary value by 1, wrapping around to 0. DO NOT CARRY DIGITS.
  • Otherwise, don't modify the digits.
  • If after incrementing, any 3-digit ternaries are 000, increment all the digits in 000 by 1 again.

Convert the ternary numbers back into letters using the Ternary Table. You should now have a decrypted word.
